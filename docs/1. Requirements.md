# ClinicHub - Requirements

## Project Overview

ClinicHub is a web application designed to manage a medical clinic. The system allows administrators, doctors, and patients to manage appointments, medical records, and exams through a centralized platform.

---

# Functional Requirements

## FR01
The system must allow administrators to register patients.

## FR02
The system must allow administrators to register doctors.

## FR03
The system must allow administrators to register medical specialties.

## FR04
The system must allow patients to schedule appointments.

## FR05
The system must allow admins and doctors to cancel appointments.

## FR06
The system must allow doctors to view their scheduled appointments.

## FR07
The system must allow doctors to create medical records.

## FR08
The system must allow doctors to request exams.

## FR09
The system must allow patients to view their requested exams and results.

## FR10
The system must authenticate users before granting access.

---

# Non-Functional Requirements

## NFR01
The backend must be developed using Java and Spring Boot.

## NFR02
The database must use MySQL.

## NFR03
The frontend must consume REST APIs.

## NFR04
Passwords must be encrypted.

## NFR05
The project must be containerized using Docker.

## NFR06
The system must follow RESTful principles.

## NFR07
The application must support CRUD operations for all main entities.
