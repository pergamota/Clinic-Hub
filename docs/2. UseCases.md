# ClinicHub - Use Cases

## UC01 - User Login

Actor:
- User

Description:
Allows a registered user to access the system.

Flow:
1. User enters email.
2. User enters password.
3. System validates credentials.
4. User is redirected to the dashboard.

--------------------------------------------------

## UC02 - Register Patient

Actor:
- Administrator

Description:
Registers a new patient in the system.

Flow:
1. Administrator fills in patient information.
2. System validates CPF and email.
3. System creates a User account.
4. System creates the Patient profile.

--------------------------------------------------

## UC03 - Register Doctor

Actor:
- Administrator

Description:
Registers a new doctor.

Flow:
1. Administrator enters doctor's information.
2. System validates CRM.
3. System creates User.
4. System creates Doctor.

--------------------------------------------------

## UC04 - Register Specialty

Actor:
- Administrator

Description:
Creates a new medical specialty.

Flow:
1. Administrator enters specialty name.
2. System validates if it already exists.
3. System saves the specialty.

--------------------------------------------------

## UC05 - Schedule Appointment

Actor:
- Patient

Description:
Schedules a medical appointment.

Flow:
1. Patient selects a doctor.
2. Patient selects an available date.
3. System checks availability.
4. Appointment is created.

--------------------------------------------------

## UC06 - Cancel Appointment

Actor:
- ADMIN, Doctor

Description:
Cancels an appointment.

Flow:
1. Admin or Doctor selects appointment.
2. Admin or Doctor informs cancellation reason.
3. Appointment status becomes CANCELLED.

--------------------------------------------------

## UC07 - View Doctor Schedule

Actor:
- Doctor

Description:
Displays all appointments assigned to the doctor.

Flow:
1. Doctor logs in.
2. Dashboard displays today's appointments.

--------------------------------------------------

## UC08 - Create Medical Record

Actor:
- Doctor

Description:
Creates the medical record after a consultation.

Flow:
1. Doctor opens appointment.
2. Doctor fills diagnosis.
3. Doctor fills prescription.
4. Doctor adds notes.
5. Medical record is saved.

--------------------------------------------------

## UC09 - Request Exam

Actor:
- Doctor

Description:
Requests one or more exams.

Flow:
1. Doctor opens appointment.
2. Doctor selects exam type.
3. System creates exam request.

--------------------------------------------------

## UC10 - View Exam Results

Actor:
- Patient

Description:
Displays requested exams and their results.

Flow:
1. Patient logs in.
2. Patient accesses Exams page.
3. System displays exam status and results.
